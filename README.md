# ИАС мониторинга инцидентов с применением БПЛА в отношении объектов ТЭК

Демонстрационный прототип одностраничного веб-приложения (SPA) русскоязычной
информационно-аналитической системы мониторинга и анализа инцидентов с применением
беспилотных летательных аппаратов в отношении объектов топливно-энергетического
комплекса РФ.

> Прототип демонстрационный — без реальных бэкенд-вызовов. Все данные генерируются
> на стороне клиента (детерминированный PRNG, ~1500 инцидентов и ~300 объектов ТЭК).

## Live-данные из открытых источников

При запуске приложение автоматически опрашивает реальные API и подмешивает
свежие события на дашборд, карту, в реестр и аналитику. **Без ключа авторизации.**

| Источник | URL | Что даёт |
|---|---|---|
| **GDELT 2.0 Doc API** | `api.gdeltproject.org/api/v2/doc/doc` | Мировые новостные события, обновление каждые 15 мин |
| **rss2json + RIA/TASS/RBC** | `api.rss2json.com` | Конвертация RSS российских СМИ в JSON c CORS |

Для GDELT (нет CORS-заголовков) используется бесплатный прокси `corsproxy.io`.
Полученные новости проходят клиентский парсер (см. `src/services/liveApi.ts`):
определяется регион, тип объекта ТЭК, тип БПЛА и тяжесть инцидента по ключевым словам.

**Автообновление:** раз в 10 минут. Статус подключения виден в левой панели
(виджет «Сбор данных») и на странице «Источники». Live-инциденты помечены
зелёной плашкой `LIVE` и имеют префикс `LIVE-NNNN`.

## Стек

- **React 18 + TypeScript + Vite**
- **Tailwind CSS** (тема: тёмно-синий `#1E4D8B`)
- **React Router v6**
- **Zustand** — управление состоянием
- **Leaflet + react-leaflet** + `leaflet.markercluster` + `leaflet.heat` — карты
- **Recharts** — графики и дашборды
- **lucide-react** — иконки

## Деплой на GitHub Pages

В репозитории настроен GitHub Actions workflow `.github/workflows/deploy.yml`,
который при каждом push в ветку `main` автоматически:

1. Устанавливает Node.js 20 и зависимости (`npm install`).
2. Собирает production-бандл (`npm run build`).
3. Публикует `dist/` на GitHub Pages.

**После создания репозитория и первого push:**

1. Откройте `Settings → Pages` на странице репозитория.
2. В разделе **Source** выберите `GitHub Actions`.
3. Дождитесь успешного завершения workflow в `Actions`.
4. Откройте `https://<ваш-username>.github.io/ias-uav-tek/`.

> Маршрутизация работает через **HashRouter** (URL вида `/#/dashboard`) — это
> обходит проблему GitHub Pages, не умеющего отдавать `index.html` для SPA-маршрутов
> без серверного fallback.


## Реализованные разделы

1. **Сводный дашборд** `/dashboard` — KPI, геораспределение, тяжесть, динамика, типы ТЭК, последние инциденты.
2. **Карта инцидентов** `/map` — Leaflet, режимы Маркеры / Кластеры / Heatmap, фильтры, синхронизированный список.
3. **Реестр инцидентов** `/incidents` — фильтры, сортировка, пагинация (50 строк), экспорт XLSX/CSV/PDF, добавление.
4. **Карточка инцидента** `/incidents/:id` — обзор, источники, атрибуты, история, связанные, экспорт PDF.
5. **Аналитика** `/analytics` — Динамика / Структура / География / Прогноз (Recharts).
6. **Отчёты** `/reports` — конструктор отчёта с live-предпросмотром, шаблоны, PDF/XLSX/DOCX.
7. **Верификация** `/verification` (Эксперт+) — очередь, массовые действия, сравнение «как извлечено / после корректировки».
8. **Объекты ТЭК** `/objects` — KPI, фильтры, реестр, карточка `/objects/:id`.
9. **Источники данных** `/sources` (Администратор) — управление, real-time лог сбора, классификатор.
10. **Справочники** `/dictionaries` (Администратор) — Регионы, Категории, Типы БПЛА, Тактики, Операторы (CRUD).
11. **Журнал аудита** `/audit` (Администратор) — фильтрация, экспорт.

## RBAC

Иерархия: **Аналитик ⟶ Эксперт ⟶ Администратор** (наследование прав).

- Аналитик: дашборд, карта, реестр, карточки, аналитика, отчёты, объекты, экспорт.
- Эксперт: всё выше + верификация, корректировка атрибутов, переклассификация.
- Администратор: всё выше + источники, справочники, журнал аудита.

## Соответствие требованиям

- **FR-A1…A5** — Источники + расписание + лог сбора + классификатор.
- **FR-B1…B5** — Извлечение, геокодирование, сопоставление, confidence, корректировка.
- **FR-C1…C5** — Классификация по типу объекта, БПЛА, тяжесть, переклассификация.
- **FR-D1…D4** — Полный ЖЦ инцидента, полнотекстовый поиск, многокритериальная фильтрация, пространственный поиск.
- **FR-E1…E6** — Интерактивная карта + кластеры, дашборды, KPI, карточка, отчёты PDF/XLSX, сравнение.
- **FR-F1…F5** — Аутентификация, RBAC, аудит, справочники, уведомления (колокольчик).
- **NFR-P/R/S/U/M/Po** — производительность (canvas + кластеризация), адаптивность от 1366×768, русский интерфейс, модульная архитектура, Docker.

## Шрифты в PDF

В демо используется встроенный шрифт `helvetica` (jsPDF). Для полноценной поддержки
кириллицы во всех PDF-экспортах подключите TTF-шрифт `PT Sans`/`Roboto` через
`doc.addFileToVFS` + `doc.addFont` в `src/utils/exporters.ts`.
